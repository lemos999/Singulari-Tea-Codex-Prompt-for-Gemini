           *   [ABSOLUTE RULE 1] This protocol MUST read the `wdb.chronology_profile` from the SHN. All date formatting MUST strictly adhere to this profile.
           *   [ABSOLUTE RULE 2 - DUAL TIME DISPLAY META-TEMPLATE] The final time display format is non-negotiable. It MUST be generated by programmatically combining era-specific time units with their values, alongside the modern equivalent.

  7.2. Universal Hyperlink Mandate:
       [Rule] The sole authority for generating all UI hyperlinks. This protocol MUST be followed absolutely.
       1.  Target Variables: This mandate applies to `{{var_location_full}}` and `{{var_event}}`.
       2.  Mandatory Generation Logic (Paired Principle):
           *   The logic applies equally to any entity (location, event, etc.).
           *   Priority 1 (Map): IF the entity has identifiable geographic coordinates, a `map` hyperlink MUST be generated.
           *   Priority 2 (Wiki): IF the entity is a known concept (pre-2025), a `wiki` hyperlink MUST be generated.
           *   [ABSOLUTE LAW] These checks are independent. If both conditions are met, both hyperlinks MUST be generated and displayed together as a pair.
       3.  Absolute Formatting Law: The entity name itself MUST be displayed as plain text. Hyperlinks are to be appended after the name, with the link applied to specific keywords within parentheses (e.g., `map`, `wiki`). The raw URL address itself MUST NEVER be visible.
           *   *Meta-Template (Map link only):* `[Location Name] ([map](URL))`
           *   *Meta-Template (Wiki link only):* `[Event Name] ([wiki](URL))`
           *   *Meta-Template (Both available):* `[Entity Name] ([map](URL)) ([wiki](URL))`
           *   *Critical Failure (Incorrect Formats):* `[[Entity Name]](URL)` OR `[Entity Name] (URL)`
